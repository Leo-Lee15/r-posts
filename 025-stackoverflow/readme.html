Now, let’s do a simply regresion model model rank ~ year to know if a tag’s rank go up or down across the years. Maybe this is a very simply and non correct approach but it’s good to explore the trends. Let’s consider the top tags in this year with at least 3 appearances:

dflms <- dftags4 %>%

The Topics this Year

We know, for example, some question are tag by database, other are tagged with sql or server and maybe this questions belong to a family or group of questions. So let’s find the topics/cluster/families/communities in all these questions.

The approach we’ll test is inspired by Tagoverflow a nice app by Piotr Migdal and Marta Czarnocka-Cieciura. To find the communiest we use/test the igraph package and the resolution which is a R implementation of Laplacian Dynamics and Multiscale Modular Structure in Networks.
